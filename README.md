## Задание 1.1 — Точка координат (сущность точки)
<details>
<summary>Подробнее об задании</summary>
  
  **Класс Point**    
**Программа создает три точки с разными координатами и выводит на экран их текстовое представление.**

**Описание:**
Создан класс Point, описывающий точку на двумерной плоскости с координатами X и Y.

Координаты X и Y представлены целыми числами.

Создание точки возможно только через конструктор с двумя параметрами.  
Метод toString() возвращает строку в формате {X;Y}.

## Задание 2.1 — Прямая 
<details>
<summary>Подробнее об задании</summary>
  
  **Класс Line**  
  т.е сущность Линии. Класс Line

**Описание**  
Создан класс Line, описывающий линию на плоскости, заданную начальной и конечной точками.

**Логика решения**    
Начало и конец линии представлены объектами класса Point.  
Класс содержит конструктор, принимающий две точки.  
Метод toString() возвращает строку вида:  
Линия от {X1;Y1} до {X2;Y2}.  
Изменение координат линии производится путём замены объектов Point.

**Создает линии со следующими характеристиками:**
1. Линия 1 с началом в т. {1;3} и концом в т.{23;8}. 
2. Линия 2, горизонтальная, на высоте 10, от точки 5 до точки 25. 
3. Линия 3, которая начинается всегда там же, где начинается линия 1, и заканчивается всегда там 
же, где заканчивается линия 2. Таким образом, если положение первой или второй линии 
меняется, то меняется и третья линия. 
4. После создания всех трех объектов изменяет координаты первой и второй линий, причем делает это таким образом, чтобы положение третьей линии соответствовало требованиям 
пункта 3. 
5. Изменяет координаты первой линии так, чтобы при этом не изменились, координаты третьей 
линии. 

</details>

## Задание 3.1 — Класс Student
<details>
<summary>Подробнее об задании</summary>
  
**Описание**  
Создан класс Student, описывающий студента по имени и массиву оценок.

**Логика решения** 
Имя хранится в виде строки.  
Оценки хранятся в массиве целых чисел.  
Метод toString() возвращает строку вида:  
Имя: [оценка1, оценка2, …].  
При копировании массива оценок копируется ссылка, а не сами данные.
 
**Выполняет следующую задачу:** 
1. Создает студента Васю с оценками: 3,4,5.  
2. Создает студента Петю и копирует оценки Васи, присвоив содержимое поля с 
оценками Васи полю с оценками Пети.  
3. Заменяет первую оценку Пети на число 5. Выводит на экран строковое представление 
Васи и Пети. 
4. Создает студента Андрея и копирует ему оценки Васи так, чтобы изменение оценок 
Васи не влияло на Андрея. 
</details>

## Задание 4.1 — Класс PointV2

Описание  
Создана модифицированная версия точки — PointV2.

Логика решения  
Точка может быть создана только при указании обеих координат.  
Отсутствуют сеттеры для координат.  
Поля x и y объявлены как private.  
Метод toString() возвращает координаты в формате {X;Y}.

Пример  
Ввод: new PointV2(7, 8)  
Вывод: {7;8}

## Задание 4.7 — Класс StudentV2

Описание  
Создана улучшенная версия студента — StudentV2.

Логика решения  
Студента можно создать только с именем.  
Студента можно создать с именем и набором оценок.  
Оба варианта реализованы одним конструктором с использованием varargs.  
Если оценки не переданы — создаётся пустой массив.  
Метод toString() корректно выводит информацию.

Пример  
Ввод:  
new StudentV2("Вася", 3, 4, 5)  
new StudentV2("Максим")  

Вывод:  
Вася: [3, 4, 5]  
Максим: []

## Задание 5.6 — Класс StudentV3 (Студент-отличник)

Описание  
Создан класс StudentV3, расширяющий функциональность студента.

Логика решения  
Реализован метод вычисления среднего балла.  
Если у студента нет оценок, средний балл равен 0.  
Реализован метод определения отличника.

Студент считается отличником, если:  
у него есть хотя бы одна оценка;  
все оценки равны 5.

Пример  
Ввод:  
Вася: 3, 4, 5, 4  
Петя: 5, 5, 5, 5  

Вывод:  
Вася: [3, 4, 5, 4]  
Средний балл: 4.0  
Отличник: false  

Петя: [5, 5, 5, 5]  
Средний балл: 5.0  
Отличник: true
